# 코드 리뷰 스킬 (verify-code)

PR 또는 커밋 전 코드 품질을 종합적으로 점검합니다. 구현이 완료된 코드를 대상으로 아래 6개 영역을 순서대로 검토하고 발견된 이슈를 보고합니다.

---

## 실행 시점

- 기능 구현 완료 후 커밋·PR 전
- 다른 사람의 코드를 리뷰할 때
- `rules-workflow` Step 14 (코드 품질 검토) 수행 시
- "코드 좀 봐줘", "리뷰해줘" 요청 시

---

## Step 0: 리뷰 범위 확정

인자가 주어진 경우 해당 파일/기능에 집중한다. 없으면 `git diff HEAD` 또는 `git diff main` 기준으로 변경된 파일 목록을 확인한 후 리뷰 범위를 사용자에게 보고하고 진행한다.

```bash
git diff --name-only HEAD
```

---

## Area 1: 로직 정확성

- 요구사항·기능 목적 대비 구현이 올바른지 확인한다.
- 조건문·분기 처리에서 누락된 케이스(null, undefined, 빈 배열, 0 등)를 점검한다.
- 비동기 처리(async/await, Promise)에서 에러 핸들링이 누락된 경로가 없는지 확인한다.
- 체크:
  - [ ] 핵심 시나리오가 의도대로 동작하는가?
  - [ ] 엣지 케이스(빈 값, 경계값, 예외 입력)가 처리되었는가?
  - [ ] 에러 핸들링이 빠진 비동기 경로가 없는가?

---

## Area 2: 타입 안전성

- `any` 사용 여부를 확인하고 `unknown` 또는 구체 타입으로 대체 가능한지 검토한다.
- 런타임 입력(API 응답, 폼 데이터, URL 파라미터)에 Zod 등 검증 로직이 적용되었는지 확인한다.
- 타입 단언(`as`)이 근거 없이 사용된 경우를 지적한다.
- 체크:
  - [ ] `any` 사용이 정당화되었는가?
  - [ ] 외부 입력에 런타임 검증이 적용되었는가?
  - [ ] 불필요한 타입 단언(`as`)이 없는가?

---

## Area 3: 중복 및 재사용성

- 동일하거나 유사한 로직이 여러 곳에 복사되어 있는지 확인한다.
- 기존 유틸·훅·컴포넌트로 대체 가능한 새 구현이 있는지 점검한다.
- 한 함수/컴포넌트가 두 가지 이상의 책임을 갖는지 확인한다.
- 체크:
  - [ ] 중복 구현이 없는가?
  - [ ] 기존 코드를 재사용할 수 있는 경우가 없는가?
  - [ ] 단일 책임 원칙이 지켜지는가?

---

## Area 4: 사이드 이펙트 및 회귀

- 변경된 코드가 다른 기능·라우트·공유 상태에 의도치 않은 영향을 주는지 확인한다.
- 전역 상태, Context, 공유 훅, 공통 유틸을 수정한 경우 영향 범위를 열거한다.
- DB 쿼리·API 호출에 side effect(쓰기, 삭제)가 의도치 않게 추가되지 않았는지 확인한다.
- 체크:
  - [ ] 변경이 다른 기능에 영향을 주는 경우가 없는가?
  - [ ] 공유 코드 수정 시 영향 범위를 파악했는가?
  - [ ] 의도치 않은 쓰기/삭제 동작이 없는가?

---

## Area 5: 가독성 및 네이밍

- 변수·함수·컴포넌트 이름이 역할을 명확히 반영하는지 확인한다.
- 마법 숫자(magic number), 하드코딩된 문자열이 상수 또는 설정으로 분리되었는지 점검한다.
- 함수 길이가 과도하게 길어 분할이 필요한 경우를 지적한다.
- 체크:
  - [ ] 이름만 보고 역할을 알 수 있는가?
  - [ ] 마법 숫자·하드코딩된 값이 상수로 분리되었는가?
  - [ ] 한 함수가 100줄을 넘지 않는가?

---

## Area 6: 불필요 코드 및 정리

- `console.log`, `console.error` 등 디버그 출력이 남아 있는지 확인한다.
- 사용하지 않는 `import`, 변수, 함수, 주석 처리된 코드 블록을 제거 대상으로 표시한다.
- TODO/FIXME 주석 중 이번 작업에서 해결되지 않은 것을 목록화한다.
- 체크:
  - [ ] `console.*` 출력이 제거되었는가?
  - [ ] 미사용 import·변수가 없는가?
  - [ ] 주석 처리된 dead code가 없는가?

## 보고 형식

리뷰 결과는 다음 형식으로 보고한다:

```
## 코드 리뷰 결과

### PASS 항목
- (문제 없는 영역 요약)

### 개선 필요 항목
| 위치 | 영역 | 내용 | 심각도 |
|------|------|------|:------:|
| src/foo.ts:42 | 타입 안전성 | API 응답에 any 사용 | 중 |
| src/bar.tsx:15 | 불필요 코드 | console.log 미제거 | 낮음 |

### 수정 권장 우선순위
1. [높음] ...
2. [중] ...
3. [낮음] ...
```

심각도 기준: **높음** (배포 차단 수준) / **중** (품질 저하) / **낮음** (권장 사항)
